Running a Postgres database locally

Jan 1, 2018 â€¢ 1 min read

In this tutorial, I'll show how you can run a Postgres (also known as PostgreSQL) database locally and connect to it using pgAdmin. We'll run the database using a Docker container.

1. Run the Docker container

First we need to run the postgres container:

docker run --name postgres -d -p 5432:5432 -e POSTGRES_PASSWORD=postgres -e POSTGRES_USER=postgres -e POSTGRES_DB=employees -v ${PWD}/postgres-docker:/var/lib/postgresql/data postgres

Let's deconstruct this docker command:

docker run --name postgres [OPTIONS] postgres

  • Run a container called postgres using the postgres image.

-d

  • Run as a detached container, so it runs in the background of your terminal

-p 5432:5432

2. Connect to the Postgres database 

from sqlalchemy import create_engine 

db=create_engine("postgresql://postgres:postgres@localhost:5432/employees")

bootstrap_sql = """

CREATE TABLE COMPANY(
   ID INT PRIMARY KEY     NOT NULL,
   NAME           TEXT    NOT NULL,
   AGE            INT     NOT NULL,
   ADDRESS        CHAR(50),
   SALARY         REAL,
   JOIN_DATE      DATE
);

INSERT INTO COMPANY (ID,NAME,AGE,ADDRESS,SALARY,JOIN_DATE) VALUES (1, 'John', 32, 'London', 20000.00,'2001-07-13');
INSERT INTO COMPANY (ID,NAME,AGE,ADDRESS,SALARY,JOIN_DATE) VALUES (2, 'David', 25, 'Dublin', 30000.00, '2007-12-13');
INSERT INTO COMPANY (ID,NAME,AGE,ADDRESS,SALARY,JOIN_DATE) VALUES (3, 'Sarah', 25, 'Edinburgh', 40000.00, '2007-12-13');
"""

with db.connect() as con:
    
    try:

        rs = con.execute(bootstrap_sql)
